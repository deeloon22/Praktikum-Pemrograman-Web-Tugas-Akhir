### Deskripsi
Modul ini merupakan bagian dari praktikum Pemrograman Web yang berfokus pada pengenalan dan penerapan Version Control System (VCS) menggunakan **Git** dan **GitHub**. Tujuan utamanya adalah agar mahasiswa memahami alur kerja pengelolaan proyek web secara profesional menggunakan sistem kontrol versi.

### Tujuan Praktikum
- Memahami konsep dasar *version control system (VCS)*.  
- Menguasai operasi dasar Git: `init`, `add`, `commit`, `push`, `pull`, dan `merge`.  
- Mampu membuat dan mengelola repository lokal dan remote (GitHub).  
- Mampu berkolaborasi dan menyelesaikan konflik saat pengembangan proyek.  

### Tinjauan Pustaka
Version Control System (VCS) adalah sistem yang merekam setiap perubahan pada berkas proyek sehingga versi sebelumnya dapat diakses kembali.
Jenis-jenis VCS meliputi:
1. Local VCS â€“ menyimpan versi file di komputer lokal.
2. Centralized VCS â€“ menggunakan server pusat (contoh: SVN).
3. Distributed VCS â€“ setiap pengguna memiliki salinan penuh repository (contoh: Git).
Git dikembangkan oleh Linus Torvalds pada tahun 2005. Git menggunakan pendekatan snapshot, bukan sekadar menyimpan daftar perubahan file. Hal ini membuat Git sangat cepat dan efisien, serta menjamin integritas data melalui hash SHA-1.
